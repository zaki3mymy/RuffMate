# Ruff Configuration Management Web App - 要件定義書

## プロジェクト概要

Pythonのlintツール「ruff」のルール設定を簡単に管理できるウェブアプリケーションの開発。

### 背景・目的
- ruffには800以上のルールがあり、手動での設定管理が困難
- プロジェクトごとにルール設定を再利用したい
- `select = ["ALL"]` + `ignore = ["ルールコード"]` 方式での設定管理を効率化
- 設定変更時のコメント付与による管理性向上

## 機能要件

### 1. データ取得・管理機能
- **データソース**: ruff公式ドキュメント（https://docs.astral.sh/ruff/rules/）
- **取得方式**: 公式ドキュメントの解析（スクレイピング）
- **更新頻度**: アプリケーション起動時に自動更新
- **データ永続化**: ブラウザのローカルストレージ

### 2. ルール管理機能

#### 2.1 ルール表示
- **基本情報表示**: ルールコード、名前、ON/OFF状態
- **詳細情報表示**:
  - ルールの詳細説明
  - コード例・修正例
  - 公式ドキュメントのLegend情報（stable/deprecated/preview、自動修正可否）
- **カテゴリ別表示**: ルールをカテゴリごとにグループ化表示

#### 2.2 ルール操作
- **個別ルール操作**: 各ルールの有効/無効切り替え
- **カテゴリ一括操作**: カテゴリ単位での一括有効/無効切り替え
- **コメント付与**: 無効化したルールに対する理由コメント追加

#### 2.3 検索・フィルタ機能
- **テキスト検索**: ルール名・説明での検索
- **カテゴリフィルタ**: 特定カテゴリのみの表示
- **状態フィルタ**: 有効/無効ルールのみの表示
- **Legend フィルタ**: stable/deprecated/preview による絞り込み

### 3. 設定出力機能

#### 3.1 pyproject.toml生成
- **基本形式**: `select = ["ALL"]` + `ignore = ["ルールコード"]`
- **コメント付与**: 各ignoreルールに理由コメントを自動追加
- **形式例**:
```toml
[tool.ruff.lint]
select = ["ALL"]
ignore = [
    "D203",  # D211 と競合するため除外
    "E501",  # 行長制限をプロジェクト固有に調整
]
```

#### 3.2 設定管理
- **エクスポート**: 現在の設定をpyproject.toml形式で出力
- **インポート**: 既存のpyproject.tomlから設定を読み込み
- **プリセット**: 用途別（Web開発、データ分析等）の設定テンプレート提供

## 技術要件

### アーキテクチャ
- **構成**: SPA（Single Page Application）
- **フロントエンドのみ**: バックエンド不要の構成
- **データ処理**: ブラウザ内でのデータ取得・処理・保存

### 技術スタック
- **Core Framework**: React 18 + TypeScript
- **ビルドツール**: Vite
- **状態管理**: Zustand（軽量で直感的）
- **UIライブラリ**: Material-UI v5
- **テストフレームワーク**:
  - ユニット：Vitest + Testing Library
  - E2E：Playwright
- **コード品質**: ESLint + Prettier
- **型安全性**: TypeScript strict mode

### パフォーマンス要件
- **大量データ対応**: 800+ルールの効率的表示（仮想スクロール）
- **レスポンシブデザイン**: モバイル・タブレット・デスクトップ全対応
- **検索性能**: リアルタイム検索・フィルタリング
- **初期読み込み**: 3秒以内での初期表示完了

## 品質要件

### テスト要件
- **カバレッジ目標**: 100%（全関数・分岐・条件を網羅）
- **テスト種別**:
  - ユニットテスト：全サービス・コンポーネント・ユーティリティ
  - 結合テスト：状態管理・データフロー
  - E2Eテスト：主要ユーザーシナリオ
- **テスト自動化**: CI/CDパイプラインでの自動実行

### コード品質
- **型安全性**: TypeScript strict mode でのエラーゼロ
- **Linting**: ESLint ルールの完全遵守
- **Formatting**: Prettier による統一フォーマット
- **依存関係**: 脆弱性のない最新パッケージの使用

## デプロイ・運用要件

### デプロイ方式（複数対応）
1. **静的ホスティング**: Netlify、Vercel、GitHub Pages
2. **ローカル実行**: `npm run dev` による開発環境
3. **Dockerコンテナ**: ポータブルな実行環境

### 運用要件
- **更新容易性**: ruffルールの新規追加・変更への自動対応
- **設定互換性**: 異なるruffバージョンでの設定互換性維持
- **エラーハンドリング**: データ取得失敗時の適切なフォールバック

## ユーザビリティ要件

### 使いやすさ
- **直感的UI**: ルールの状態が視覚的に分かりやすい
- **効率的操作**: 大量ルールの快適な閲覧・操作
- **学習コストの低さ**: ruff初心者でも使いやすい設計

### アクセシビリティ
- **キーボードナビゲーション**: マウス不要での全操作
- **スクリーンリーダー対応**: ARIA属性の適切な実装
- **色覚多様性**: 色情報に依存しない情報伝達

## 制約事項

### 技術制約
- **CORS制限**: 外部ドキュメント取得時のブラウザ制限対応
- **ローカルストレージ限界**: 大量データの効率的保存
- **ブラウザ互換性**: モダンブラウザ対応（IE非対応）

### 運用制約
- **オフライン制限**: 初回データ取得にはインターネット接続必須
- **更新頻度**: ruff公式の更新頻度に依存
- **バックアップなし**: ローカルストレージのみでのデータ保存
